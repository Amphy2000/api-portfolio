# SMS Shield & Phone Validator — Developer Integration Guide & Best Practices

Verify phone number format, identify countries, detect VoIP / virtual carrier networks, and check numbers against blocklists of public temporary Receive-SMS pools. The **SMS Shield & Phone Validator API** helps developers safeguard user signups and block SMS verification fraud.

---

## 1. Architectural Overview

The API processes number validation and risk assessments locally with high efficiency:

```
[ User Inputs Phone Number ]
             |
             v
[ Sanitizer & Normalizer ] ----> Formats number to E.164 (strips spaces/brackets, prepends '+')
             |
             v
[ Country Decoder ] -----------> Resolves country code, name, and ISO code from E.164 prefix
             |
             v
[ Disposable Blocklist Check ] -> Matches number against known public burner phone receptors
             |
             v
[ VoIP Prefix Range Check ] ---> Scans prefixes for virtual routing providers (Twilio, Skype, etc.)
             |
             v
[ Risk Score & Recommender ] --> Generates security recommendation (Allow, Flag, Block) and risk score
             |
             v
[ Standardized JSON Response ]
```

---

## 2. API Reference & Parameters

### Endpoint
`GET /sms-shield`

### Request Parameters
| Parameter | Type | Required | Default | Description |
| :--- | :--- | :--- | :--- | :--- |
| `phone` | String | **Yes** | — | The target phone number to validate (e.g. `+12015550199` or `+447451234567`). Supported formats: E.164 or basic digits. |

## 3. Code Integration Examples

### Node.js (Fetch)
```javascript
async function validatePhone(phoneNumber) {
  const apiKey = 'YOUR_RAPIDAPI_KEY';
  const url = `https://disposable-virtual-phone-sms-shield.p.rapidapi.com/sms-shield?phone=${encodeURIComponent(phoneNumber)}`;

  const response = await fetch(url, {
    method: 'GET',
    headers: {
      'x-rapidapi-key': apiKey,
      'x-rapidapi-host': 'disposable-virtual-phone-sms-shield.p.rapidapi.com'
    }
  });

  const result = await response.json();
  return result; // contains carrier type, country, risk score, and recommendation
}
```

### Python (Requests)
```python
import requests

def validate_phone(phone_number):
    url = "https://disposable-virtual-phone-sms-shield.p.rapidapi.com/sms-shield"
    querystring = {"phone": phone_number}
    headers = {
        "x-rapidapi-key": "YOUR_RAPIDAPI_KEY",
        "x-rapidapi-host": "disposable-virtual-phone-sms-shield.p.rapidapi.com"
    }
    
    response = requests.get(url, headers=headers, params=querystring)
    return response.json()
```

## 4. Key SMS Shield Best Practices

1.  **Blocking Registration Fraud (SaaS Signups):**
    Before dispatching expensive One-Time Passwords (OTPs) via SMS providers like Twilio or Vonage, check the number using this API:
    *   **If `security.recommendation` is `Block` (Risk Score: 100)**: Instantly reject the signup. These numbers are public, meaning any user can read the verification code sent to them.
    *   **If `security.recommendation` is `Flag / Verification Required` (Risk Score: 40–75)**: Force the user to verify via email first or pass a captcha.
2.  **Mitigating VoIP Overage Abuse:**
    VoIP numbers are cheap and can be generated by bots in bulk. Restricting signups from `VoIP / Virtual` carrier types helps eliminate automated spam account creation and protects your marketing metrics.
3.  **Local Format Normalization:**
    The API formats numbers into E.164 and returns a formatted string under `formatted`. Use this display string in your admin dashboards for standardized logging.
