# URL Shortener & Link Analytics — Developer Integration Guide & Best Practices

Create clean, branding-friendly short links, perform fast redirects, and collect comprehensive click analytics. The **URL Shortener & Link Analytics API** stores slug mappings and records visit metadata (referrers, browsers, and timestamps) using a secure connection to a Supabase PostgreSQL backend.

---

## 1. Architectural Overview

The API exposes endpoints for link creation, client redirection, and statistical analytics tracking:

```
[ Shorten Request: POST /shorten ]
                 |
                 v
      Generate 6-char Slug
                 |
                 v
   Save to Supabase `short_links`
                 |
                 v
        Return Short URL JSON

--------------------------------------------------

[ Redirect Request: GET /redirect?slug=xyz ]
                 |
                 v
       Fetch Target original_url
                 |
                 +---> (Asynchronously) -> Increment clicks in `short_links`
                 +---> (Asynchronously) -> Log metadata into `short_link_clicks`
                 |
                 v
    Execute `302 Redirect` to Target

--------------------------------------------------

[ Analytics Request: GET /shorten?slug=xyz ]
                 |
                 v
     Fetch click records from DB
                 |
                 v
  Compile total clicks, referrers
  breakdown, and browser distribution
                 |
                 v
      Return Analytics JSON Board
```

---

## 2. API Reference & Parameters

### 1. Create a Short URL
*   **Method:** `POST`
*   **Path:** `/shorten`
*   **Body Type:** `application/json`
*   **Request Body:**
    *   `url` (String, Required): The target long URL to shorten. Must start with `http://` or `https://`.

### 2. Redirect Visitor
*   **Method:** `GET`
*   **Path:** `/redirect`
*   **Query Parameters:**
    *   `slug` (String, Required): The 6-character identifier (e.g. `jrfx5u`).

### 3. Get Link Analytics
*   **Method:** `GET`
*   **Path:** `/shorten`
*   **Query Parameters:**
    *   `slug` (String, Required): The 6-character identifier (e.g. `jrfx5u`).

## 3. Code Integration Examples

### Node.js - Shorten a Link (POST)
```javascript
async function createShortUrl(longUrl) {
  const apiKey = 'YOUR_RAPIDAPI_KEY';
  const url = 'https://url-shortener-link-analytics.p.rapidapi.com/shorten';

  const response = await fetch(url, {
    method: 'POST',
    headers: {
      'content-type': 'application/json',
      'x-rapidapi-key': apiKey,
      'x-rapidapi-host': 'url-shortener-link-analytics.p.rapidapi.com'
    },
    body: JSON.stringify({ url: longUrl })
  });

  const result = await response.json();
  return result; // contains slug, original_url, short_url
}
```

### Python - Fetch Analytics (GET)
```python
import requests

def get_link_analytics(slug):
    url = "https://url-shortener-link-analytics.p.rapidapi.com/shorten"
    querystring = {"slug": slug}
    headers = {
        "x-rapidapi-key": "YOUR_RAPIDAPI_KEY",
        "x-rapidapi-host": "url-shortener-link-analytics.p.rapidapi.com"
    }
    
    response = requests.get(url, headers=headers, params=querystring)
    return response.json()
```

### cURL - Redirect Endpoint
```bash
curl --request GET \
	--url 'https://url-shortener-link-analytics.p.rapidapi.com/redirect?slug=jrfx5u' \
	--header 'x-rapidapi-host: url-shortener-link-analytics.p.rapidapi.com' \
	--header 'x-rapidapi-key: YOUR_RAPIDAPI_KEY'
```

## 4. Supabase Database Table Configuration

To activate the persistent production database, execute the following script in the **Supabase SQL Editor** under your project dashboard:

```sql
-- Create short_links table
CREATE TABLE short_links (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    slug TEXT UNIQUE NOT NULL,
    original_url TEXT NOT NULL,
    clicks BIGINT DEFAULT 0,
    created_at TIMESTAMPTZ DEFAULT NOW()
);
CREATE INDEX idx_short_links_slug ON short_links(slug);

-- Create short_link_clicks table for analytics logging
CREATE TABLE short_link_clicks (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    link_id BIGINT REFERENCES short_links(id) ON DELETE CASCADE,
    referrer TEXT,
    user_agent TEXT,
    clicked_at TIMESTAMPTZ DEFAULT NOW()
);
CREATE INDEX idx_clicks_link_id ON short_link_clicks(link_id);
```

Configure your Vercel project environment settings with `SUPABASE_URL` and `SUPABASE_SERVICE_ROLE_KEY` to link your deployment to the database.
